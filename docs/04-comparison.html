Chapter 4

Same score, different models

All three finetuned models score 75% verdict accuracy on the 100 held-out questions. If you stopped there you'd conclude the two alignment methods did nothing. They did plenty — just not to the headline number. This chapter measures what each one actually changed, question by question.

How far each method moved from SFT

Compare each aligned model to SFT one question at a time and the two methods look nothing alike. DPO rewrote most of the answers and overturned a chunk of verdicts. GRPO barely budged — its KL leash (Chapter 3) held it close to SFT.

GRPO's correct-answer set is in fact identical to SFT's — same 75 questions right, same 25 wrong, just reworded on 40 of them. DPO's is not: it swapped 5 questions in for 5 others, netting zero.

DPO drifted toward "yes"

When DPO rewrote answers, it didn't do so evenly. The reference set is 60 yes / 31 no / 9 maybe, and SFT roughly tracks that. DPO tilted the whole model toward answering yes — a well-known side effect of preference optimization shifting the output distribution. GRPO kept SFT's balance.

The shared blind spot: every model scores 0 of 9 on maybe. It's the rarest class — 90 of 800 training examples (11%), and just 9 of the 100 test items — and none of SFT, DPO, or GRPO ever gets one right. With only 9 test cases treat this as a qualitative flag, not a precise rate; still, this, not the yes/no split, is where more data or a class-aware reward would help most.

How to read this

This is a compact educational demonstration, not a benchmark. Before generalizing any number above:

  • One run, no seeds or error bars. At n=100, a 75% rate has a 95% interval of roughly 66–84%. "No accuracy change" means not detectable here, not proven equal. The maybe class is n=9 — its 0/9 is a flag, not a measurement.
  • Verdict-only metric. Accuracy scores just the leading yes/no/maybe token; it says nothing about whether the rationale is faithful — exactly where DPO's rewriting might help or hurt.
  • The two methods optimized different things. GRPO's reward is verdict correctness; DPO's preference was only "look like the curated answer, not raw base output." So comparing them on verdict accuracy is partly apples-to-oranges — DPO was never really asked to fix verdicts.

Takeaways

  1. SFT did the heavy lifting. Base → SFT is 0% → 75%; the base model never even emits a verdict. Alignment starts from an already-capable model.
  2. Here, RL didn't move the headline. On a 7B model with a small budget and a verdict-only metric, SFT had already extracted what a short run recovers. Read that as a fact about this setup, not a law about RL.
  3. The signal shapes the change. DPO (a preference) rewrote style and shifted the distribution; GRPO (a verifiable reward under a KL leash) stayed anchored. Same score, different failure modes — the real lesson.
  4. Where to push next: the maybe class (0/9) — more maybe data, a class-balanced reward, more GRPO steps at a looser leash, or DPO pairs built from wrong-verdict rejections instead of raw base output.
