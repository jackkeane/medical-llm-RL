This one takes that finetuned model and runs the next stage — preference alignment — two different ways: DPO and GRPO. Then it asks the honest question: did either actually help? Where alignment sits in the lifecycle

An LLM is rarely finetuned once and shipped. There is a pipeline, and alignment is the middle stage — you teach the format (SFT), then teach which answers are preferred (RL), then shrink the model for deployment (compression). You align before compressing, so that pruning and quantization preserve the aligned behavior.

The result in one picture

Here is the headline, and it is a deliberately humbling one. SFT takes the base model from never producing a verdict at all to 75% correct. Then both alignment methods… hold at 75%. The lesson isn't that RL is useless — it's that RL is not where accuracy comes from on a small run like this, and that the two methods change the model in very different ways underneath an identical score. Chapter 4 opens that up.
